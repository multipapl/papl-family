# Family Canvas

Интерактивное семейное дерево для семьи.

## Что это за продукт

Это не “редактор графа”.
Это mobile-first семейное приложение, в котором обычный родственник:

1. находит человека;
2. открывает его карточку;
3. переходит к предкам, потомкам или своей ветке;
4. только при необходимости открывает карту дерева;
5. при необходимости включает отдельный режим редактирования.

## Для кого проект

Проект делается в первую очередь:

- для пожилых родственников;
- для обычных членов семьи;
- для телефона и планшета;
- для спокойного, понятного сценария без технических терминов.

Главный приоритет всегда такой:

1. понятно пожилому человеку;
2. удобно с телефона;
3. только потом красиво и технологично.

## Текущий этап

Сейчас проект находится на этапе подготовки первой публичной беты.

Цель этой итерации:

- выложить живой сайт в интернет;
- отправить ссылку семье;
- собрать реальный фидбек по удобству;
- при этом не усложнять систему настоящим backend раньше времени.

## Что уже есть в коде

Текущий локальный прототип уже умеет:

- искать человека;
- открывать карточку;
- показывать блоки `Родители / Партнеры / Дети`;
- переключать режимы `Моя ветка / Предки / Потомки / Все дерево`;
- показывать карту семьи;
- двигать и масштабировать карту на тач-экране;
- локально редактировать карточки и добавлять родственников в рамках текущей сессии.

## Что является следующим шагом

Следующая задача проекта:

- сделать публичную бету на `Vercel`;
- добавить локальное сохранение в браузере через `localStorage`;
- оставить редактирование доступным пользователю;
- при этом не захардкодить архитектуру под `JSON`.

## Актуальная архитектурная идея

Сейчас данные берутся из локального `JSON`, но интерфейс не должен навсегда зависеть от него.

Правильное направление:

- UI работает с доменной моделью;
- доступ к данным идет через repository-слой;
- сейчас используется `JSON` как seed и локальное сохранение в браузере;
- позже это заменяется на `Supabase` без полной переделки интерфейса.
